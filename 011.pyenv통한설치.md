일반적으로 Python 3.11.6을 설치하는 방법은 시스템 환경과 목적에 따라 여러 가지가 있지만, **IT 전문가의 개발 환경**에서는 시스템을 오염시키지 않고 버전 관리를 유연하게 할 수 있는 **`pyenv`를 통한 설치**를 가장 권장합니다.

아래에서는 세 가지 일반적인 방법을 비교하고, 가장 효율적인 **`pyenv` 방식**을 상세히 안내해 드립니다.

## 1\. 🐍 Python 설치 방법 비교

| 방법 | 장점 | 단점 | 권장 상황 |
| :--- | :--- | :--- | :--- |
| **`apt` (패키지 관리자)** | 가장 빠르고 쉽다. | Ubuntu 저장소의 버전이 **최신이 아닐 수 있다.** 시스템 Python과 섞여 환경 오염 위험이 있다. | 단순 테스트 환경 또는 특정 레거시 버전 설치 시. |
| **공식 Installer/Homebrew** | 비교적 간편하다. | 여러 버전 관리(전환)가 어렵고, **시스템 환경 변수($\text{PATH}$)를 수동으로 관리**해야 함. | 단일 버전만 사용하는 개인 PC. |
| **`pyenv` (버전 관리자)** | **버전 격리**가 완벽하고, 프로젝트별 버전 전환이 자동화된다. | 설치 과정에 빌드 도구가 필요하고, 초기 설정이 약간 복잡하다. | **전문적인 개발 환경** (다중 프로젝트, 3vm구축실습 환경). |

## 2\. 🥇 `pyenv`를 통한 Python 3.11.6 설치 (권장)

`pyenv`를 사용하여 Python을 설치하는 것은 현재 진행하시는 프로젝트 환경에 가장 적합한 방법입니다.

### A. 사전 준비: 빌드 도구 설치 및 `pyenv` 확인

Python 소스를 컴파일하는 데 필요한 빌드 도구를 설치하고, `pyenv`가 정상적으로 설치되었는지 확인합니다.

```bash
# 1. 빌드 도구 설치 (Ubuntu/Debian 기준)
sudo apt update
sudo apt install -y build-essential libssl-dev zlib1g-dev libbz2-dev \
    libreadline-dev libsqlite3-dev wget curl llvm libncurses5-dev \
    libncursesw5-dev xz-utils tk-dev libffi-dev liblzma-dev python3-openssl

# 2. pyenv 초기화 확인
source ~/.bashrc  # (pyenv 설정이 적용되도록)
pyenv --version   # 버전 정보가 출력되어야 합니다.
```

### B. Python 3.11.6 설치 실행

`pyenv install` 명령을 사용하여 원하는 버전을 설치합니다.

```bash
pyenv install 3.11.6
```

> **주의:** 이 명령어는 소스 코드를 다운로드하고 컴파일하므로, 시스템 성능에 따라 \*\*상당한 시간(5분 이상)\*\*이 소요될 수 있습니다.

### C. 설치 완료 확인

설치 후 `pyenv versions`로 목록을 확인합니다.

```bash
pyenv versions
```

설치된 목록에 `3.11.6`이 나타나면 성공입니다.

### D. 프로젝트 환경에 활성화

설치된 버전을 특정 프로젝트 폴더에 활성화하여 **격리된 환경**을 만듭니다.

```bash
# 1. 프로젝트 폴더로 이동
cd ~/3vm구축실습예제/flask_project

# 2. 해당 폴더에서 3.11.6 사용 지정
pyenv local 3.11.6

# 3. 확인 (이제 'python' 명령은 3.11.6을 가리킵니다)
python --version
```

## 3\. 🥈 `apt`를 통한 설치 (빠르지만 덜 유연함)

`apt`를 사용하면 설치는 빠르지만, 버전이 구버전일 수 있고 프로젝트별 전환이 어렵습니다.

```bash
# 1. 패키지 목록 업데이트
sudo apt update

# 2. Python 3.11 설치 (Ubuntu 저장소에 해당 버전이 있어야 함)
# Ubuntu 버전에 따라 패키지명이 다를 수 있습니다.
sudo apt install python3.11 python3.11-venv
```

이후 `python3.11` 명령어를 사용하여 실행해야 하며, `pyenv`처럼 **`python`** 명령어를 자동으로 전환해 주지는 않습니다.

**pip**은 Python으로 작성된 소프트웨어 패키지들을 설치하고 관리하는 데 사용되는 **패키지 관리 시스템(Package Management System)**입니다.

---

## 1. 🥇 pip의 정의와 역할

* **P**ackage **I**nstaller for **P**ython의 약자로 흔히 알려져 있습니다. (재귀 약자($\text{Recursive Acronym}$)로 $\text{pip}$ $\text{installs}$ $\text{Packages}$라고도 불립니다.)
* **주요 역할:** Python 패키지 저장소인 **PyPI (Python Package Index)**에 등록된 수많은 외부 라이브러리(예: Flask, Django, Pandas, NumPy 등)를 사용자의 Python 환경에 다운로드하고 설치합니다.
* **표준화:** Python 3.4 버전(CPython)부터 **기본적으로 포함**되어 제공되므로, 별도의 설치 없이 바로 사용할 수 있는 표준 도구입니다.

---

## 2. ⚙️ 핵심 사용법 및 명령어

| 명령어 | 역할 | 예시 |
| :--- | :--- | :--- |
| **설치** | PyPI에서 지정된 패키지를 다운로드하고 설치합니다. | `pip install requests` |
| **제거** | 설치된 패키지를 시스템에서 제거합니다. | `pip uninstall requests` |
| **목록 확인** | 현재 활성화된 환경에 설치된 모든 패키지 목록을 보여줍니다. | `pip list` |
| **환경 추출** | 현재 환경의 모든 패키지와 버전을 파일로 저장합니다. (가장 중요) | `pip freeze > requirements.txt` |
| **파일 설치** | 저장된 `requirements.txt` 파일을 읽어 필요한 패키지들을 한 번에 설치합니다. | `pip install -r requirements.txt` |
| **업그레이드** | 설치된 패키지를 최신 버전으로 업데이트합니다. | `pip install --upgrade requests` |

## 3. ⚠️ [Expert Tip] pip 사용 시 주의사항 (환경 격리)

**3vm구축실습예제**와 같은 전문적인 환경에서 **`pip`는 반드시 격리된 환경에서만 사용해야 합니다.**

### 1. 전역(Global) 환경 사용 금지
* `pyenv`나 `venv`를 사용하지 않고 `pip install`을 실행하면, 해당 패키지가 **시스템 전체**의 Python 환경에 설치됩니다.
* 이는 **다른 프로젝트와의 패키지 버전 충돌**을 야기하고, 최악의 경우 **운영체제가 사용하는 기본 Python 기능(시스템 Python)을 오염시켜 시스템 안정성을 해칠 수 있습니다.**

### 2. 가상 환경(Virtual Environment) 사용 필수
* **모범 사례:** `pyenv`로 **Python 버전**을 격리하고, 그 안에서 **`python -m venv [가상환경이름]`** 명령을 통해 프로젝트별로 **패키지 환경**을 격리해야 합니다.
* `pip` 명령어는 반드시 가상 환경을 **`source [가상환경이름]/bin/activate`**로 **활성화**한 상태에서 사용해야 합니다.

### 3. 종속성 명세화 (requirements.txt)
* 프로젝트에 어떤 패키지가 필요한지 **`requirements.txt`** 파일에 명확히 명세해야 합니다. 이 파일은 프로젝트를 **Docker 컨테이너**로 만들거나, 다른 개발자에게 공유할 때 **환경을 동일하게 재현**하기 위한 필수 문서입니다.

```
